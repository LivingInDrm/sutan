# 雪中悍刀行 Web 卡牌游戏设计规范 v1

> 适用范围：`src/renderer/ui/` 全部游戏界面、组件与后续 AI 重构任务  
> 主题方向：**水墨北凉**  
> 版本性质：**第一版规范**，用于统一现有视觉语言，并为后续组件重构提供明确标准

---

## 1. 核心设计原则

### 1.0 世界观融入原则（Diegetic UI）

本项目所有 UI 设计必须满足一个前置条件：**界面中的每一个容器、控件、文案与反馈，都必须像存在于《雪中悍刀行》的世界中，而不是漂浮在游戏之上的现代产品界面。**

#### 1.0.1 所有 UI 容器必须是世界观内的物件

- 所有容器都应被理解为某种可触碰、可命名、可归属的物件，例如：**卷轴、竹简、木牌、宣纸、信函、账簿、令牌、案台、军报、卷宗**
- 不能出现现代白色卡片、圆角面板、阴影浮层等 SaaS / Dashboard 风格元素
- 面板即使承担信息分组作用，也必须先有“物件身份”，再谈布局与可读性
- 用户第一眼应感受到“正在翻阅旧卷、查看军报、拈起命牌”，而不是“正在操作产品首页”

#### 1.0.2 文案必须使用世界观内的语言

- 文案必须优先采用叙事式、沉浸式、世界观内的说法
- 不能直接出现产品化说明文字，例如“支持继续游戏”“导入导出存档”“开发版”等现代软件表达
- 同样含义应改写为游戏世界中的表达，例如：
  - “支持继续游戏” → “旧卷尚在，提笔可续”
  - “导入存档” → “引旧卷入案”
  - “导出存档” → “誊录此卷”
- 所有说明文字都应像卷首题记、批注、路引、军报，而不是帮助文档

#### 1.0.3 交互隐喻必须来自古代

- 按钮应像**牌匾、印章、令牌、木牍**
- 选项应像**签文、命牌、卦象、批签**
- 列表应像**名册、卷宗、簿册**
- 进度应像**沙漏、香烛、刻痕、军令时限**
- 不能照搬现代网页交互隐喻，例如 toggle、checkbox、progress bar、pricing table

#### 1.0.4 面板材质必须有纹理

- 禁止纯白 / 纯色实底面板
- 所有面板必须具备明确材质感，可用宣纸纤维、木纹、竹节、绢丝、旧皮革、铜扣、墨晕、旧金边框来构建
- 面板可以是浅色，但不能是干净的白色卡片；必须像“旧纸”“陈卷”“案上文书”
- 边缘处理优先考虑木框、铜扣、压边、卷轴轴头、纸张毛边，而非现代圆角 + 阴影盒子
- 背景图只负责材质质感，例如纹理、纤维、自然边缘、毛边，不承载结构性线条与边框
- 标题分隔线、信息块层次、边界节奏等结构关系必须由 CSS 控制，不得烘焙进背景图
- 这样即使 `background-size` 拉伸，内容位置仍可通过 `padding / margin / gap` 精确控制，不让素材反向绑架布局

#### 1.0.5 禁止出现的 SaaS 模式

- 禁止定价表式的功能对比
- 禁止 Feature list 式说明文字
- 禁止 Dashboard 状态卡片
- 禁止 Landing page 左右分栏布局
- 即使使用左右结构，也必须服务于“摊开的画卷 / 书案陈设 / 主副卷面”的叙事构图，而不是网页产品销售页

## 2. 视觉主题

### 2.1 主题定义

本项目不是现代科技产品界面，而是一款以《雪中悍刀行》为题材的武侠叙事卡牌游戏。整体视觉应呈现：

- **宣纸褪色感**：像旧纸、旧卷轴、旧军报，而不是高饱和数字界面
- **水墨灰黑感**：墨迹、炭灰、乌金，层次靠材质与浓淡变化建立
- **北凉边塞感**：肃杀、苍凉、克制，带一点铜锈、旧金和风沙感
- **朱砂点睛感**：危险、命运、抉择、行刑倒计时等才允许使用红色强调
- **Q版写意素材感**：角色立绘是 Q 版卡通水墨，地图是水墨山水，物品是写意水墨，不应出现写实欧美 UI 语汇

### 2.2 氛围关键词

- 宣纸
- 墨迹
- 北凉
- 边塞
- 旧金
- 铜绿
- 朱砂
- 卷轴
- 江湖
- 克制

### 2.3 界面观感目标

- **第一眼**：像一本摊开的武侠绘卷，而不是后台管理系统
- **第二眼**：信息可读、层次稳定、颜色语义统一
- **第三眼**：交互反馈像“落印、落笔、翻页、投牌”，而不是网页默认 hover

### 2.4 素材风格统一要求

#### 角色立绘

- 采用 **Q版卡通水墨人物**，轮廓清晰、头身比偏夸张、衣纹保留水墨笔触
- 背景必须弱化或透明，避免复杂环境抢占卡面主信息
- 角色主色可有差异，但整体饱和度控制在中低范围

#### 地图与场景

- 地图使用 **水墨山水 + 北凉边塞地貌**
- 地点图标需像插画印记或手绘路引，不得像现代游戏 minimap icon
- 场景背景允许有雾、风沙、墨晕，但不得用赛博霓虹、强镜面高光

#### 物品与装备

- 倾向 **写意水墨静物**，强调轮廓、材质、器物神韵
- 武器、酒葫芦、戒指、飞剑等物件应保留“旧器物”气质
- 避免塑料感、金属镜面感、现代图标风

#### UI 素材使用规则

- UI 素材优先使用完整图，不使用九宫格切片
- 只有当组件存在明确、长期、可复用的拉伸需求时，才例外评估九宫格方案
- 当 CSS 难以自然表达目标材质、轮廓、边缘起伏时，优先使用完整生图背景
- CSS 的职责应聚焦于状态、排版、交互反馈与结构控制，而不是勉强模拟复杂材质

### 2.5 现有风格基线

当前项目已有可继承的正向设计决策：

- 颜色基础已建立：`leather / parchment / gold / crimson / ink`
- 字体基础已建立：`LXGW WenKai` 用于展示，`LXGW WenKai Lite`（缺失时回退 `LXGW WenKai` / `Source Han Serif CN`）用于正文与 UI
- 材质基础已存在：`rice-paper`、`ink-wash`、`bronze`、`silk`
- 结算页 `ui_004` 边框、宣纸底与书卷式布局已具备正确方向

后续所有重构必须围绕上述基线统一，而不是重新引入新的审美系统。

## 3. 色彩体系

### 3.1 设计原则

- **背景色暖，文字色稳，强调色少**
- 不使用纯黑 `#000000` 和纯白 `#FFFFFF` 作为主界面底色
- 不使用现代默认蓝、紫、荧光绿作为主要状态色
- 用 **金 / 竹青 / 青墨 / 朱砂** 建立游戏语义色

- 行刑倒计时：朱砂系

- 右侧叙事纸面：`#d4c5a9`
- 左侧投入区：深铜/深墨底 + 边框图案
- 结果色：
  - 大成功：亮金 + 浅竹青辅助
  - 成功：竹青
  - 部分成功：黄铜金
  - 失败：朱砂
  - 大失败：深朱砂

### 3.5 阴影与辉光颜色

沿用现有 shadow token，并限定使用场景：

- `--shadow-gold-sm`: 小型 hover、高亮字
- `--shadow-gold`: 激活按钮、章节标题
- `--shadow-gold-lg`: 仅用于高价值卡、标题屏主视觉
- `--shadow-danger`: 危险提示
- `--shadow-rarity-*`: 仅用于稀有度与卡牌品质，不滥用

### 3.6 禁止项

- 禁止 `blue-*` 作为核心品牌交互色
- 禁止 `green-*` 直接作为完成色
- 禁止大面积纯红、纯黄、纯白高饱和覆盖
- 禁止在同一面板中混入超过 1 个强调色 + 1 个状态色
- 禁止为了“更丰富”而叠加无助于叙事、层级与可读性的装饰
- 角块装饰、朱印点缀、额外徽记若不承担明确的信息职责，应直接删除

## 4. 排版规则

### 4.1 字体体系

| 角色 | Token | 字体 |
|---|---|---|
| 展示字体 | `--font-display` | `LXGW WenKai`, `STKaiti`, `KaiTi`, serif |
| 正文字体 | `--font-body` | `LXGW WenKai Lite`, `LXGW WenKai`, `Source Han Serif CN`, serif |
| UI字体 | `--font-ui` | `LXGW WenKai Lite`, `LXGW WenKai`, `Source Han Serif CN`, serif |
| 等宽字体 | `--font-mono` | `Source Code Pro`, `Consolas`, monospace |

### 4.2 使用规则

- **标题、地名、场景名、按钮主标签**：优先用 `--font-display`
- **剧情叙事、卡牌说明、段落正文**：使用 `--font-body`
- **数值、HUD、小型状态标签**：使用 `--font-ui`
- **资源数字、调试信息、计数器**：使用 `--font-mono`

### 4.3 字号层级

以下是游戏专属字号，不照搬 Apple 体系。

| 角色 | 字号 | 字重 | 行高 | 字距 | 字体 | 用途 |
|---|---:|---:|---:|---:|---|---|
| 标题画面主标题 | 56px | 400~700 | 1.08 | 0.12em | display | `雪中悍刀行` |
| 屏幕级标题 | 36px | 700 | 1.15 | 0.08em | display | 地点标题、章节标题 |
| 场景标题 | 28px | 700 | 1.2 | 0.06em | display | 场景页标题 |
| 面板标题 | 22px | 700 | 1.25 | 0.04em | display | 卡牌详情、结算区块 |
| 卡牌名称 | 18px | 700 | 1.25 | 0.02em | display / body | 卡牌主名 |
| 正文大段 | 16px | 400 | 1.8 | 0.02em | body | 剧情叙事 |
| 正文常规 | 14px | 400 | 1.7 | 0.01em | body | 描述、说明 |
| UI标签 | 12px | 500~700 | 1.5 | 0.08em | ui | 标签、状态、次要说明 |
| 微型说明 | 10px | 500 | 1.4 | 0.12em | ui | 计数、角标、辅助信息 |
| 极小数值 | 9px | 600 | 1.2 | 0.08em | ui / mono | 卡牌角标、微型标识 |

### 4.4 中文排版原则

- 中文正文禁止过紧；正文行高应在 **1.65~1.9**
- 展示标题可适度拉开字距，但不要超过 `0.12em`
- 正文不使用全大写风格；英文缩写只用于内部技术字段或极少数 UI 标记
- 段落宽度尽量控制在 **18~28 个汉字/行** 的舒适阅读范围
- 武侠叙事正文优先左对齐，不做大面积居中排版

### 4.5 排版风格建议

#### 标题

- 用毛笔感展示字体，但要控制使用范围
- 一屏内最多 1 个强展示标题
- 同一区块不同时使用 3 种以上字号

#### 正文

- 用宋体系统承载叙事，形成书卷感
- 卡牌正文、场景叙事、结算文本风格要统一

#### 数字

- 日数、金币、回合、成功数等可用 `font-mono`，增强信息清晰度
- 但资源大标题不得显得像程序后台，应配合 display/ui 字体做组合

## 5. 组件样式

本章定义的是**游戏专属组件样式**，不是通用互联网产品控件。

### 5.1 角色卡 / 装备卡 / 物品卡

#### 尺寸

- 标准卡：`192px ~ 208px` 宽，纵横比约 `1 : 1.55`
- 手牌紧凑卡：`112px × 192px`
- 卡牌圆角：`8px`
- 内边距：标准卡 `12px`，紧凑卡 `6px`

#### 结构

1. 顶部卡名区
2. 立绘/物品图主体区
3. 类型 / 稀有度 / 标签区
4. 描述区
5. 属性区（人物/装备专属）

#### 样式

- 背景：宣纸纹理 + 轻微旧纸渐变
- 上沿可使用 `2px` 金色或稀有度渐变高光线
- 卡牌边框必须表达品质，但不使用炫彩霓虹
- hover 动效：
  - 放大 `1.04~1.06`
  - 上移 `4~8px`
  - 阴影略增强
- selected 态：
  - `2px` 亮金外环
  - 可附加轻微金辉
- locked 态：
  - 降到 `opacity: 0.45~0.55`
  - 可加 `grayscale(0.2~0.35)`

#### 稀有度映射

| 稀有度 | 边框 | 底光 | 标签色 |
|---|---|---|---|
| 传说 / 金 | `#c9a84c` | 金辉 | 金底深字 |
| 史诗 / 铜 | `#8a6d2b` | 铜辉 | 铜金底浅字 |
| 稀有 / 银 | `#c4b594` | 冷纸光 | 宣纸底深字 |
| 凡品 / 石 | `#5a3520` | 极弱阴影 | 深皮革底浅字 |

### 5.2 手牌区

#### 布局

- 高度固定：`208px`（现有实现 `h-52`）
- 左侧分类标签栏宽：`48px`
- 分类分隔线：`1px`
- 卡牌展示区左右安全边距：`20px`

#### 样式

- 背景：深皮革底 + 低透明度墨纹覆盖
- 顶部分割光：`1px` 宣纸色渐变线
- 分类标签用竖排 display 字体，激活态亮度提高
- 超出时边缘使用左右渐隐，而不是原生滚动条

#### 行为

- 双击手牌投入槽位
- 单击手牌打开详情
- 被占用/已锁定卡牌在手牌中保持可见，但视觉降权

### 5.3 结算面板

#### 总体结构

- 使用书卷/水墨边框作为总容器
- 左侧：投入卡牌与结算操作
- 右侧：叙事文本与结算结果
- 中线分隔：`1px` 暗金分割线

#### 样式参数

- 外层面板建议宽高：`95% × 92%` 视口
- 内容安全内边距：
  - 横向：`8%`
  - 纵向：`10%`
- 右侧标题：
  - 14px~18px display
  - 字色用 `#1a0f0a`
  - 跟随细金分割线

#### 结果表现

- 成功：竹青标题 + 金色辅助线
- 失败：朱砂标题 + 暗红提示
- 骰子区、结果区、叙事区必须有明显阅读顺序

### 5.4 骰子组件

当前骰子仍有现代默认红绿蓝问题，重构时统一如下：

#### 尺寸

- 标准骰：`48px × 48px`
- 大骰：`56px × 56px`
- 圆角：`8px`
- 边框：`2px`

#### 配色

| 状态 | 背景 | 边框 | 文字 |
|---|---|---|---|
| 成功 | `rgba(61,85,37,0.55)` | `#8ab06a` | `#e8dcc8` |
| 普通失败 | `rgba(107,15,15,0.40)` | `rgba(139,26,26,0.6)` | `#d44040` |
| 爆骰 | 深金底 | 亮金环 | `#f5f0e8` |
| 重投 | 青墨底 | 浅青墨环 | `#e8dcc8` |

#### 动效

- 入场：缩放 + 轻旋转
- 爆骰：亮金 pulse，不使用黄色默认 glow
- 重投：青墨 ring，不使用亮蓝 ring

### 5.5 地图节点

#### 结构

- 节点图标
- 外发光 / 状态环
- 地点名称牌签

#### 样式

- 图标尺寸：`96px ~ 112px`
- 可参与节点：
  - 外发光：柔和金色
  - 外环：亮金 pulse
- 已清空节点：
  - 竹青弱辉光
  - 标签亮度下降
- 名称牌签：
  - 背景：半透明黑或深皮革
  - 字体：display
  - 高度：`20px ~ 24px`
  - 内边距：`2px 8px`

### 5.6 场景界面

#### 左右分栏

- 左：背景图 + 槽位区
- 右：宣纸信息面板
- 左右建议比例：`1:1`

#### 场景卡片

- 可参与：金系边框 + hover 响应
- 进行中：青墨标签
- 已完成：竹青标签
- 未解锁：深皮革低对比卡面 + 解锁提示

#### 槽位组件

- 形态类似“牌位/卡槽/印位”
- 必填槽位应有清晰提示
- 空槽位边框使用暗金，hover 使用亮金
- 已放入卡牌后要显著比空槽更稳定

### 5.7 标题画面

#### 结构

1. 全屏背景
2. 主标题
3. 副标题 / 世界观短句
4. 命牌式选择卡签
5. 版本号

#### 样式规范

- 主标题必须使用 display 字体
- 主标题颜色：主金 `#c9a84c`
- 副标题颜色：宣纸色 60% 或旧金弱态
- 选择项应实现为命牌 / 签文式卡签，而不是统一按钮列；强调“拈牌定命”的选择感
- 标题可读性优先通过 `text-shadow`、背景局部压暗、标题附近轻微明暗控制解决，不强制外加深色玻璃容器
- 主操作按钮只保留「启此新卷 / 续写前卷」这一层级，其余入口可退化为文字链接
- 背景应从“纯渐变”升级为“深皮革 + 墨色 + 微弱宣纸雾感”

### 5.8 按钮系统

#### 设计方向：牌匾风

按钮的视觉隐喻是**中国传统木质匾额 / 牌匾**，而不是现代UI的圆角矩形。
整体感觉：厚重、古朴、有历史感，像挂在府衙或武馆门口的匾额。

#### 高层原则

- 当 CSS 难以自然表达目标材质、轮廓与边缘时，优先使用完整生图背景
- CSS 负责状态、排版、尺寸、交互反馈与结构控制，不勉强承担复杂材质模拟
- 次要操作可退化为文字链接，不强制所有可点击项都做成实体按钮

#### 形状与结构

- **圆角**：`2~3px`（几乎直角），绝不能用 `8px` 以上的现代圆角
- **双层边框**：外层粗描边（`2px`，暗金 `#8a6d2b`）+ 内层细描边（`1px`，亮金 `#c9a84c`），形成匾额的"边框套边框"感
- **角钉装饰**：四角各有一个小圆点或菱形装饰（`4~6px`），模拟匾额上的铜钉/铆钉，颜色 `#c9a84c`
- **横梁感**：按钮上下边缘可以比左右边缘略粗（上下 `2px`，左右 `1.5px`），暗示木质横梁结构

#### 纹理与质感

- **背景**：优先使用完整按钮生图承载材质、边缘起伏与轮廓，不再以“深金渐变 + 铜质纹理”CSS 方案作为默认解法
  - 主按钮：完整牌匾 / 题签式背景，质感接近旧金、旧木或题卷压边
  - 次按钮：可降级为更克制的深木 / 深皮革题签，或直接退为文字链接
  - 危险按钮：使用完整深朱砂 / 旧漆质感底图，避免现代警报按钮感
- **状态变化**：hover / active / disabled 主要通过亮度、对比、阴影、位移与字色变化表达，而不是重新绘制另一套结构
- **避免**：纯色平涂、光滑线性渐变、金属高光反射，以及用大量 CSS 细节硬模拟复杂材质

#### 文字处理

- 字体：`--font-display`（霞鹜文楷），书卷感书写体
- 字色：主按钮用亮宣纸 `#f5f0e8`（深底上亮字），次按钮用金 `#c9a84c`
- 字间距：`0.1~0.15em`，匾额刻字通常有较大字距
- 文字应有微弱的 `text-shadow`（`0 1px 2px rgba(0,0,0,0.5)`），模拟刻字的凹陷感

#### 主按钮 (primary)

- 背景：深金渐变 `#6a5020` → `#8a6d2b`，叠加轻微铜质纹理
- 双层描边：外 `#8a6d2b` / 内 `#c9a84c`
- 四角铜钉装饰
- 字色：`#f5f0e8`
- hover：整体微亮（铜钉和边框提亮到 `#f0d060`），`box-shadow` 增加金色外发光
- active：`scale(0.98)` + 阴影收缩，像按下匾额

#### 次按钮 (secondary)

- 背景：深皮革 `#2a1810` → `#3d2418`
- 单层描边：暗金 `#8a6d2b`
- 无角钉（比主按钮朴素）
- 字色：`#c9a84c`
- hover：边框微亮

#### 危险按钮 (danger)

- 背景：深朱砂 `#4a0e0e` → `#6b0f0f`
- 描边：暗朱砂 `#8b1a1a`
- 字色：`#f5f0e8`
- 用于放弃、删除等危险操作，慎用

#### 幽灵按钮 (ghost)

- 无背景、无边框
- 金色文字 `#c9a84c`
- hover：出现轻微底色 `rgba(138,109,43,0.1)`
- 用于次要操作（返回、取消）

#### 选择按钮 (choice)

- 用于玩家抉择场景（三选一等）
- 宣纸底 `#d4c5a9` + 暗金单层边框
- 字色：墨色 `#1a0f0a`
  - 字体：`--font-body`（霞鹜文楷轻量版，缺失时回退 `LXGW WenKai` / `Source Han Serif CN`），叙事感
- 选中态：边框变亮金 + 左侧出现朱砂竖条标记
- 布局：宽条形，文字左对齐

#### 尺寸

| 尺寸 | padding | 字号 | 最小宽度 | 用途 |
|------|---------|------|----------|------|
| sm | `6px 16px` | `13px` | `80px` | 标签、工具栏 |
| md | `10px 24px` | `15px` | `120px` | 大部分场景 |
| lg | `14px 32px` | `17px` | `160px` | 重要操作 |
| xl | `18px 40px` | `20px` | `200px` | 标题页主按钮 |

#### 禁用态

- 不是灰色遮罩，而是"墨色褪去"：整体 `opacity: 0.35`，颜色偏灰褐
- 铜钉和边框褪色
- cursor: `not-allowed`

#### 绝对禁止

- ❌ 圆角 `≥ 8px` 的现代按钮
- ❌ 光滑的线性/径向CSS渐变
- ❌ Material Design / iOS 风格的按钮
- ❌ 纯色平涂无纹理
- ❌ 默认浏览器 focus ring（用金色 outline 替代）

## 6. 布局原则

### 6.1 整体一屏布局

游戏主布局以**桌面一屏沉浸式**为核心，标准结构如下：

1. **顶部状态栏**
2. **中央主游戏区**
3. **底部手牌区**

对应现有布局方向：

- 顶部状态栏固定高度：`48px`
- 底部手牌区固定高度：`208px`
- 中央区占据剩余空间并负责地图 / 场景 / 结算 / 商店等页面内容

### 6.2 布局比例

在 `1440 × 900` 参考桌面下：

- 顶栏：`48px`
- 主区：`644px`
- 手牌区：`208px`

允许按屏幕高度同比伸缩，但**顶部与底部必须保持稳定高度感**，不能压到不可用。

### 6.3 间距系统

建议统一采用以下 spacing 级别：

| Token | 数值 | 用途 |
|---|---:|---|
| xs | 4px | 微距、图标与字 |
| sm | 8px | 小组件内距 |
| md | 12px | 卡片常规内距 |
| lg | 16px | 面板内距、控件间距 |
| xl | 24px | 区块间距 |
| 2xl | 32px | 大区块间距 |
| 3xl | 48px | 页级留白 |

### 6.4 栅格与对齐

- 主要内容区使用视觉对齐，不强调网页型栅格线
- 关键元素需沿以下对齐轴：
  - 左对齐轴：标题、描述、列表起点
  - 中轴：结算与标题屏主视觉
  - 底对齐轴：槽位区与手牌区

### 6.5 页面专属布局

#### 地图页

- 背景全屏铺满
- 顶部 HUD 浮于地图上方
- 地图名称固定在左下角
- 地点节点按地图坐标绝对定位

#### 地点页

- 顶部为地点标题与返回
- 中部为场景列表
- 背景允许使用场景大图 + 纸面遮罩

#### 场景页

- 左右分栏
- 左侧更强调沉浸感，右侧更强调信息可读性

#### 结算页

- 书卷中心布局
- 外圈保留暗背景呼吸空间，不要顶满屏幕

### 6.6 圆角规范

| 用途 | 圆角 |
|---|---:|
| 微小标签 | 4px |
| 标准卡片 / 面板 | 8px |
| 大面板 | 12px |
| 圆形按钮 / 环形装饰 | 9999px / 50% |

规则：矩形组件一般不超过 `12px`，避免现代 App 风过强。

## 7. 层级与阴影

### 7.1 层级系统

| 层级 | 名称 | 说明 |
|---|---|---|
| L0 | 地图层 | 地图背景、场景背景、材质底图 |
| L1 | 场景内容层 | 地图节点、卡槽、场景列表、主内容 |
| L2 | 固定 UI 层 | 顶部状态栏、底部手牌区、页内标题 |
| L3 | 浮动交互层 | tooltip、卡牌 hover、按钮高亮 |
| L4 | 弹出层 | 卡牌详情、剧情提示、小弹窗 |
| L5 | 模态层 | 结算阻断、重大决策、整屏叙事弹层 |

### 7.2 阴影原则

此项目的“深度”不靠现代多层投影，而靠：

- 材质差异
- 明暗对比
- 描边与纸张边缘
- 少量金辉 / 墨影

### 7.3 阴影规范

| 等级 | 样式 | 用途 |
|---|---|---|
| Level 0 | 无阴影 | 静态纸面、普通背景 |
| Level 1 | `0 2px 8px rgba(0,0,0,0.30)` | 小卡片、按钮 |
| Level 2 | `0 4px 16px rgba(0,0,0,0.40)` | 面板、浮起卡牌 |
| Level 3 | `0 8px 32px rgba(0,0,0,0.50)` | 弹层、详情、中心结算 |
| 金辉 | `0 0 20px rgba(201,168,76,0.30)` | 激活、稀有度、主强调 |
| 危险辉光 | `0 0 16px rgba(139,26,26,0.30)` | 失败与危险提醒 |

### 7.4 特殊深度表达

#### 地图层

- 不加额外阴影，靠底图本身塑造空间

#### 手牌层

- 靠顶部分割光与卡牌 hover 提升层级

#### 卡牌详情

- 使用 `backdrop-blur-sm` + 深遮罩
- 面板本体为纸面 + 稀有度辉光 + 深墨阴影

#### 模态层

- 黑色遮罩透明度 `0.55~0.70`
- 内容本体必须是唯一视觉焦点

## 8. Do's and Don'ts

### 8.1 Do

- 使用宣纸、皮革、墨色、旧金构成主界面
- 使用 `LXGW WenKai` 作为标题、地名、牌名展示字体
- 使用 `LXGW WenKai Lite` 承载剧情正文与说明文本，缺失时回退 `LXGW WenKai` / `Source Han Serif CN`
- 使用 `gold / bamboo / cerulean / crimson` 建立清晰状态语义
- 让材质成为背景层次的一部分，而不是纯色块堆叠
- 让地图、卡牌、结算三大场景共享同一套设计语言
- 让 hover 像“提笔、浮牌、落印”，克制而明显
- 让标题屏、世界地图、结算页成为风格标杆页面
- 让每一个组件都能被说成某种“物件”，而不是抽象功能模块
- 让说明文案像卷上批注、路引、军报，而不是产品说明

### 8.2 Don't

- 不要使用默认 Tailwind 蓝、绿、紫作为主要状态色
- 不要出现科技感渐变、玻璃拟态、霓虹发光边框
- 不要在同一组件里混用 `serif` 裸写、系统 sans、毛笔字体
- 不要出现 Material 风圆角矩形按钮充当主视觉按钮
- 不要让卡牌、场景卡、按钮看起来来自三个不同游戏
- 不要在深底上使用低对比灰字
- 不要把所有强调都做成金色高亮，导致层级失真
- 不要滥用 `backdrop-blur`，水墨游戏不是现代 macOS 风
- 不要出现白色圆角卡片排列成定价表
- 不要出现“支持继续游戏”这类产品化说明文字
- 不要出现纯白面板 + 灰色边框的现代信息框
- 不要直接显示 `v0.1.0 开发版` 这类破坏沉浸感的版本信息

### 8.3 常见错误示例

#### 错误 1：HUD 颜色语义混乱

- 金币用黄
- 声望用蓝
- 思考次数用紫
- 行刑用红

问题：系统来源像拼装后台，不是统一世界观。  
正确做法：统一改为 **金 / 青墨 / 朱砂 / 宣纸弱字** 体系。

#### 错误 2：默认网页按钮

- 深灰矩形
- 默认圆角
- 默认 hover

问题：与水墨卷轴边框和卡牌系统完全割裂。  
正确做法：使用游戏按钮组件或 SVG 边框底。

#### 错误 3：详情面板现代感过强

- 纯白底
- 现代阴影
- 高饱和渐变标签

问题：破坏书卷感。  
正确做法：宣纸纹理 + 暗金细线 + 有控制的稀有度辉光。

#### 错误 4：标题页做成定价表

- 白色圆角卡片排成档位选择
- 每个选项像套餐说明
- 信息组织方式类似 pricing table

问题：视觉来源是现代 SaaS 页面，不属于武侠世界中的物件。  
正确做法：改为木牌 / 签文 / 卦象式选择，让“选难度”变成“定命数”。

#### 错误 5：文案直接暴露产品属性

- “支持继续游戏”
- “导入导出存档”
- “v0.1.0 开发版”

问题：玩家会立即意识到自己在操作软件，而不是进入江湖卷宗。  
正确做法：统一改写为世界观语言，或隐藏不必要的技术信息。

### 8.4 世界观化正反例补充

- ❌ 白色圆角卡片排列成定价表 → ✅ 木牌 / 签文 / 卦象式选择
- ❌ “支持继续游戏” → ✅ “旧卷尚在，提笔可续”
- ❌ 纯白面板 + 灰色边框 → ✅ 宣纸纹理面板 + 木框 / 铜扣
- ❌ 版本号 `v0.1.0 开发版` → ✅ 不显示，或用世界观语言包装

## 9. 响应式行为

### 9.1 目标平台

- **主目标：桌面端**
- 支持不同桌面分辨率与窗口尺寸变化
- 移动端不是主设计目标，但不能完全崩坏

### 9.2 断点建议

| 断点 | 宽度 | 行为 |
|---|---:|---|
| Large Desktop | `>= 1600px` | 中央内容可放宽，保留边缘留白 |
| Standard Desktop | `1280px ~ 1599px` | 标准布局 |
| Small Desktop | `1024px ~ 1279px` | 压缩边距、缩小标题和节点 |
| Tablet Landscape | `768px ~ 1023px` | 尽量保留双栏，但缩减手牌与 HUD |
| Mobile / Narrow | `< 768px` | 允许降级为堆叠式布局，不要求完整桌面体验 |

### 9.3 响应式规则

#### 顶部状态栏

- 保持单行优先
- 当宽度不足时，优先缩短标签文案，不先缩数字

#### 手牌区

- 桌面标准高度 `208px`
- 小桌面可降至 `184px`
- 更窄时分类栏从 48px 降到 40px

#### 地图页

- 地点图标尺寸可从 `112px` 缩到 `88px`
- 地图标题与左下信息块缩小 1 级字号

#### 场景页

- `>= 1024px`：左右分栏
- `768px ~ 1023px`：左右比例可调整为 `45/55`
- `< 768px`：改为上下堆叠，先信息后槽位或先槽位后信息，择一固定

#### 结算页

- 大屏：中心书卷布局
- 小屏：减少外边距，优先保证右侧叙事区可读

### 9.4 缩放优先级

分辨率收缩时，按以下顺序压缩：

1. 页边距
2. 装饰性留白
3. 图标尺寸
4. 标题字号
5. 面板内边距

以下内容不得优先牺牲：

- 正文可读性
- 卡牌可辨识性
- 主要按钮点击区
- 状态栏数字辨识度

### 9.5 交互热区

- 所有主要点击目标最小高度：`40px`
- 按钮推荐高度：`40px ~ 48px`
- 地图节点点击区域应大于视觉图标本体至少 `8px`
- 手牌卡牌 hover/点击区域以整个卡牌为准，不做过窄热区

## 10. Agent Prompt Guide

本章用于后续 AI 批量重构组件时，快速复用规范。

### 10.1 快速规则

1. 整体主题始终是 **水墨北凉**
2. 不允许引入现代科技蓝和默认绿色
3. 标题用 `--font-display`，正文用 `--font-body`
4. 深底主文字用 `#e8dcc8`，纸面主文字用 `#1a0f0a`
5. 主强调色固定为 `#c9a84c`
6. 成功用 `#5a7a3a`，进行中用 `#2d5555`，危险用 `#8b1a1a`
7. 背景优先使用宣纸、墨迹、铜纹材质，不使用纯净 App 风
8. 组件圆角一般不超过 `12px`
9. 阴影克制使用，优先靠材质和描边建立层次

### 10.2 快速色值参考

- 主背景：`#1a0f0a`
- 深色信息层：`#1a1a2e`
- 宣纸主底：`#d4c5a9`
- 宣纸亮面：`#e8dcc8`
- 主金：`#c9a84c`
- 亮金：`#f0d060`
- 暗金：`#8a6d2b`
- 朱砂：`#8b1a1a`
- 竹青：`#5a7a3a`
- 青墨：`#2d5555`

### 10.3 组件重构 Prompt 模板

#### 模板 1：重构通用游戏面板

> 将该组件重构为“水墨北凉”风格。背景使用宣纸或深皮革材质，标题使用 `--font-display`，正文使用 `--font-body`。主色使用旧金 `#c9a84c`，正文深底文字使用 `#e8dcc8`，纸面文字使用 `#1a0f0a`。圆角控制在 `8px~12px`，边框使用暗金 `#8a6d2b`，避免现代 App 风按钮和默认蓝色交互。

#### 模板 2：重构卡牌组件

> 将该卡牌组件重构为武侠水墨卡面。卡面使用宣纸纹理叠加轻微旧纸渐变，顶部卡名区使用 display 字体，立绘保留 Q版水墨人物主体。稀有度通过边框和弱辉光表达：传说用 `#c9a84c`，稀有用 `#c4b594`，凡品用 `#5a3520`。hover 时卡牌上浮 6px、缩放 1.05，不使用霓虹、玻璃拟态或高饱和渐变。

#### 模板 3：重构顶部状态栏

> 将顶部状态栏统一为北凉 HUD 风格：背景用 `rgba(26,26,46,0.90)`，底部分隔线用暗金 `#8a6d2b` 40% 透明度。日数与金币使用金系，声望与进行中状态使用青墨系，行刑倒计时使用朱砂系。字体以 `--font-ui` 和 `--font-mono` 组合，不允许再出现随机蓝色、紫色或默认 Tailwind 状态色。

#### 模板 4：重构场景页

> 将场景页改为“左沉浸、右叙事”的书卷式布局。左侧使用场景背景图与卡槽区域，右侧使用宣纸纹理信息面板。标题使用 `--font-display`、28px、旧金色；正文使用 `--font-body`、14px~16px、行高 1.7~1.8。可参与状态用金色，进行中用青墨，已完成用竹青，未解锁用低对比皮革底与条件提示。

#### 模板 5：重构结算页

> 将结算界面重构为中心书卷式面板，保留水墨边框与宣纸叙事区。左侧展示投入卡牌与操作按钮，右侧展示剧情、骰子与结果。成功使用竹青 + 金色辅助，失败使用朱砂。整体避免现代弹窗感，阴影以深墨投影为主，视觉重点应落在故事与结果，而不是按钮。

### 10.4 输出要求

AI 重构组件时必须：

- 优先复用已有 token：`src/renderer/styles/index.css`、`src/renderer/ui/theme/tokens.ts`
- 优先复用已有组件：`Button`、`Panel`、`DecoratedFrame`、`SectionTitle`
- 不新增新的视觉主色
- 不在组件内部硬编码 `blue-* / green-* / purple-*`
- 不再写裸 `fontFamily: 'serif'`
- 尽量将颜色、阴影、字体改为 token 化表达

### 10.5 自检清单

在提交 UI 重构前，AI 必须自查：

- 是否仍然出现默认蓝、绿、紫？
- 是否仍然存在现代网页按钮风？
- 字体是否统一为 display/body/ui 三层？
- 是否遵守顶部 / 中央 / 手牌三段布局？
- 卡牌、场景、结算是否属于同一美术世界？
- 是否过度使用 glow、blur、gradient？
- 颜色语义是否统一？
- 是否坚持宁简勿繁，删掉了无助于叙事、层级和可读性的装饰？
