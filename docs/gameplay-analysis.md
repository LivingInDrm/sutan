## 1. 当前玩法完整描述

> 本节像写游戏攻略一样描述当前版本的实际玩法，让没玩过的人能完整理解。

### 1.1 游戏背景与目标

《Sutan》是一款以《雪中悍刀行》IP为皮肤的叙事策略卡牌游戏。玩家扮演北凉世子徐凤年，手握一批角色与装备卡牌，在倒计时耗尽之前完成任务、积累资源——最核心的目标只有一个：**在处刑日到来之前，把手里那张「苏丹之印」卡处理掉，否则就会被处决。**

游戏分为简单/普通/困难/噩梦四个难度，区别在于：
- 简单：21天、初始50金
- 普通：14天、初始30金
- 困难：7天、初始15金
- 噩梦：5天、初始10金

### 1.2 一次完整游戏流程（从头到尾）

#### 开始游戏

点击开始后，游戏会把**全部卡牌**（约16+张角色、6张装备、情报/宝石/苏丹卡等）全部直接发到手牌。你拥有徐凤年、温华、老黄、洪洗象、红薯、徐渭熊、徐龙象、李淳罡、陈芝豹……等几乎所有IP核心人物。

初始资源：
- 金币（取决于难度）
- 声望：50（固定）
- 回溯次数：3次（可撤销昨天的决策）
- 思考次数：每天3次（对卡牌进行特殊分析）
- 黄金骰：0枚（可花费直接增加检定成功数）
- 处刑倒计时：14天（普通难度）

#### 每日循环

游戏以"天"为单位推进，每天分三个阶段：

**黎明阶段（自动执行）**
- 场景刷新：所有注册的场景重新激活为"可参与"状态
- 思考次数重置为3次
- 系统内部更新（玩家无感知，纯后台）

**行动阶段（玩家操作）**

这是每天唯一有决策的时间窗口。玩家可以：

1. **参与场景**：在地图上查看当前可用的场景，选择一个，将角色卡和物品卡"投入"到场景的槽位里，然后等待倒计时结算。被投入的卡牌在结算前无法用于其他场景。

2. **查看手牌**：查看所有拥有的卡牌及其属性。

3. **装备管理**：将装备卡挂载到角色上，增加该角色的属性（装备有一定槽位上限）。

4. **俺寻思（ThinkSystem）**：消耗1次"思考次数"，对某张卡牌进行分析，触发思考事件（当前UI中入口不明显，实际几乎没有使用到）。

5. **进入商店**：有一个商店场景（scene_shop_001），但目前买卖逻辑是个空壳，点进去后只显示"交易完成"。

完成行动后，点击"下一天"按钮进入结算。

**结算阶段（系统执行，玩家观看/操作）**

点击"下一天"后，游戏执行以下步骤：

1. 所有active场景的剩余回合数 -1
2. 剩余回合归零的场景进入结算
3. **未参与的到期场景**：触发缺席惩罚（声望损失）
4. **已参与的到期场景**：进入叙事+检定流程

每个到期场景按顺序播放：
- 叙事文本（对话、旁白）
- 若有分支选择，展示选项
- 若有骰检，执行骰子检定
- 根据结果（成功/部分成功/失败/大失败）应用效果
- 释放被锁定的卡牌，场景消失

全部场景结算完毕后，系统检查游戏是否结束。

#### 胜利与失败

当处刑倒计时归零时：
- **手里没有苏丹之印** → 生存胜利
- **手里仍有苏丹之印** → 被处决，游戏失败

### 1.3 场景系统详解

#### 场景是什么

场景是游戏内容的核心载体，类似"任务卡"。每个场景有：
- **名称和描述**：如"权力的游戏"、"边境破茶摊・剑影初逢"
- **类型**：event（叙事事件）、challenge（挑战）、shop（商店）
- **持续时间**：1-3天后自动结算
- **槽位**：需要投入的卡牌种类和数量（如：1个必填角色槽 + 1个可选物品槽）
- **解锁条件**：如声望≥20、拥有特定标签的卡牌等
- **缺席惩罚**：如果时间到了没有参与，会扣除声望

#### 当前有哪些场景

| 场景ID | 名称 | 类型 | 难度参考 | 结构 |
|--------|------|------|---------|------|
| scene_001 | 权力的游戏 | event | social检定，目标值5 | 单段 |
| scene_002 | 沙漠穿越 | challenge | survival min检定，目标值4 | 单段 |
| scene_003 | 决斗场 | challenge | combat first检定，目标值6 | 单段 |
| scene_004 | 命运的抉择 | event | 选择 → 两条分支旁白 | 两段（无检定） |
| scene_006 | 边境破茶摊・剑影初逢 | event | 多属性链式检定 | 多段5阶段 |
| scene_007 | 芦苇荡畔・剑甲初逢 | event | 多属性链式检定+结局选择 | 多段4阶段 |
| scene_shop_001 | 装备商人 | shop | 交易（空壳） | 单段 |

另外，北凉地图有10个地点图标（清凉山王府、听潮亭、陵州城市集等），但大多数地点目前没有绑定任何场景。

#### 场景阶段与分支

复杂场景（如scene_006、scene_007）支持多阶段递进：
- 每个阶段有叙事文本
- 通过骰检结果或玩家选择，进入不同的下一阶段
- 最终阶段（is_final）结束后场景完成

以scene_006（温华相遇）为例：
```
开场阶段（魅力检定）
  → 成功/部分成功 → 救援路线（花10金）→ 继续魅力检定
  → 失败/大失败 → 调侃路线 → 体魄检定 → 饮酒闲聊
最终结算：温华（card_wenhua_sword）加入队伍，获得不同的关系标签（"知己"/"损友"/"债主"/"嗜酒"等）
```

### 1.4 卡牌系统详解

#### 角色卡

角色卡是派遣到场景的核心资源。每张角色卡有8个维度属性（0-10分）：

| 属性 | 说明 |
|------|------|
| physique（体魄） | 格斗、体力、耐久 |
| charm（魅力） | 吸引力、气度、诱惑 |
| wisdom（智慧） | 见识、谋略、感悟 |
| combat（战斗） | 实战搏斗能力 |
| social（社交） | 交际、说服、人情世故 |
| survival（生存） | 野外求生、危机应对 |
| stealth（潜行） | 隐匿、侦查、刺杀 |
| magic（法术） | 道术、灵力 |

此外还有两个特殊属性：
- **support**：为同场景其他角色提供支援骰（未见明显UI）
- **reroll**：本次检定中，可以选择几颗失败骰重投

当前游戏中有17张角色卡，主要是《雪中悍刀行》IP人物，属性设计有明显人设差异：
- 徐龙象：体魄10 / 战斗10，纯战力担当
- 徐凤年：各属性均衡，reroll最高（4），全能型
- 洪洗象：智慧10 / 法术10，谋略型
- 南宫仆射：魅力10 / 战斗10 / 潜行7，战力+颜值
- 老黄：战斗10 / support5，最高辅助+战力

#### 装备卡

装备卡挂载到角色上，为角色的特定属性提供加值。例如：
- 大凉龙雀（神品）：combat +10, charm +3, wisdom +2
- 木马牛（神品）：combat +12, physique +2, wisdom +2
- 寒玉戒指（银品）：wisdom +6, magic +5, charm +2
- 十二飞剑（金品）：combat +8, magic +10, stealth +4

装备加值较高，顶级装备的combat加值（+12）甚至超过大多数角色的本体combat值。

#### 特殊卡

- **苏丹之印**：危险卡，持有到处刑日则输
- **情报卡**（intel_001 宫廷流言）：可作为物品投入场景，提供social +3, charm +2
- **宝石**（gem_001 红宝石）：所有属性 +2
- **临时思绪卡**（temp_*）：仅在scene_006相关场景中生成，提供小型属性修正，对应不同的互动状态

#### 目前卡牌数量汇总
- 角色：17张
- 装备：6张
- 特殊（苏丹/情报/宝石）：3张
- 临时思绪卡：5张

### 1.5 检定系统详解

检定（Dice Check）是解决不确定事件的核心机制：

**骰池计算**
1. 根据场景设定的属性（如social）和计算模式（max/min/sum/avg/first）计算基础骰数
2. 加上投入角色的装备属性加成
3. 加上物品卡的属性加成
4. 最终骰数上限：20颗

**骰检规则**
- 骰子：D10（10面骰）
- 成功阈值：≥7
- 爆骰：骰出10时额外再掷1颗（可连锁，上限20颗）
- 重投：可将最多reroll值数量的失败骰重新投掷

**结果判定**

| 条件 | 结果 |
|------|------|
| 成功数 ≥ 目标值 | 成功（Success） |
| 目标值 > 2，且成功数 ≥ 目标值-2 | 部分成功（Partial Success） |
| 1 ≤ 成功数 < 目标值-2 | 失败（Failure） |
| 成功数 = 0 | 大失败（Critical Failure） |

**黄金骰**：消耗1枚直接+1个成功数，可在重投后使用。

### 1.6 回溯机制

玩家持有最多3次"回溯次数"（初始）。消耗1次可以回到昨天黎明阶段的状态：
- 金币、声望、卡牌、场景状态全部回滚
- 今天所有操作清零
- 玩家脑中的"记忆"（知道昨天发生了什么）保留

回溯让玩家可以用"先知"优势重新决策。

## 2. 各系统实现深度评估

> 对每个核心系统进行"哪些做得深/哪些只是骨架"的评估。

### 2.1 游戏主循环：⭐⭐⭐（3/5）

**做得好的部分：**
- 回合推进逻辑清晰（Dawn → Action → Settlement 三相）
- 场景生命周期管理完整（Available → Participated → Settling → Completed）
- 缺席惩罚机制存在且生效
- 时间回溯机制（rewindDay）已实现，保存了前一天的完整快照

**仅是骨架的部分：**
- 行动阶段的"玩家互动密度"非常低：主要就是点选场景，没有随机强制事件、每日诏令、势力行动
- 开局发全部卡牌，成长弧度几乎不存在
- 处刑倒计时感很弱，没有形成"必须在有限时间内处理苏丹卡"的心理压迫
- `DIFFICULTIES.initial_cards` 字段定义了但从未被使用

**代码证据：**
```typescript
// GameManager.startNewGame：把所有传入卡全部添加，不管难度
for (const card of initialCards) {
  this._allCardsMap.set(card.card_id, card);
  this.cardManager.addCard(card);
}
```

### 2.2 卡牌系统：⭐⭐⭐（3/5）

**做得好的部分：**
- CardInstance 设计简洁，支持标签系统（tags add/remove）
- EquipmentSystem 装备挂载逻辑完整，支持装备槽位限制
- CardManager 管理规范，protagonist 卡不可移除
- 角色属性区分清晰，人设差异明显

**仅是骨架的部分：**
- 角色没有长期成长（无属性提升、专长树、关系等级）
- 临时思绪卡（temp_*）是好的起点，但只在 scene_006 中出现
- 宝石系统（gem_001）定义了但无嵌入槽位机制

**装备加值偏大问题：**
顶级武器（木马牛）给 combat +12，而顶级角色本体也只有 combat 10，这会导致"拿到装备比练角色更重要"的体验，削弱IP人物辨识度。

### 2.3 场景/事件系统：⭐⭐⭐⭐（4/5）

**做得好的部分（是游戏最强的部分）：**
- Stage → Narrative → Settlement → Branch 的分层结构设计优秀
- 支持叙事、对话、效果、选择等多种节点类型
- scene_006、scene_007 已展示多阶段、多路径、属性联动的完整叙事体验
- 支持临时卡牌生成，让中间状态可以影响后续判定
- 解锁条件（声望门槛+tag门槛）的框架已建立

**仅是骨架的部分：**
- 总场景数极少：7个（含商店），其中真正有深度的只有scene_006、scene_007
- unlock_scenes 字段定义了但 EffectApplier 中未实现（场景解锁网络无法形成）
- 商店场景（scene_shop_001）执行 trade 时只返回"交易完成"文本
- 大多数地点（10个中有8个）没有绑定任何场景

**代码证据（unlock_scenes 未接通）：**
```typescript
// EffectApplier.apply：没有对 effects.unlock_scenes 做任何处理
if (effects.gold) { this.playerState.changeGold(effects.gold); }
if (effects.reputation) { ... }
// unlock_scenes 完全缺席
```

### 2.4 骰检系统：⭐⭐⭐⭐（4/5）

**做得好的部分：**
- D10 / 7+成功 / 10爆骰 的规则简洁而经典
- 爆骰连锁（最多20颗额外）增加了戏剧性
- 重投机制（reroll）为玩家提供了风险管理工具
- 黄金骰（直接+1成功）是强力保底手段
- performFullCheck 实现完整且逻辑清晰

**仅是骨架的部分：**
- 重投在UI层几乎没有明确的操作流程（玩家不清楚如何使用）
- 黄金骰获取方式不存在（初始0枚，无获取途径）
- support 属性已定义但对骰池计算没有实际影响
- CalcMode.Sum 是最有协作意义的模式，但现有场景没有使用

### 2.5 玩家状态系统：⭐⭐⭐（3/5）

**做得好的部分：**
- 5个资源维度：gold、reputation、goldenDice、rewindCharges、thinkCharges
- 声望分5个等级（Humble到Legendary），可作为解锁门槛
- 数值有合理上下限（声望0-100）

**仅是骨架的部分：**
- 金币缺少持续消耗口（无招募费、维护费、日常开销），中期后通胀严重
- 商店经济闭环未打通
- 声望只作为解锁条件，未转化为"政治资源"（影响事件池、折扣、调动等）
- thinkCharges 在 UI 中几乎没有对应入口
- rewindCharges 是强力工具，但使用时机和收益不够清晰

### 2.6 结局系统：⭐（1/5）

**做得好的部分：**
- 枚举定义了5种结局：主线胜利、生存胜利、隐藏结局、处刑失败、死亡失败

**仅是骨架的部分：**
- 实际只实现了2种：生存胜利（无苏丹卡）和处刑失败（有苏丹卡）
- 主线胜利、隐藏结局、死亡失败均无落地逻辑
- 没有角色结局、关系结局、势力结局
- 苏丹卡的"获取→处理→转嫁→消解"玩法链路未建立

**代码证据：**
```typescript

  const sultanCards = this.cardManager.getSultanCards();
  if (sultanCards.length > 0) {
    // ExecutionFailure
  } else {
    // SurvivalVictory
  }
}
```

### 2.7 地图/导航系统：⭐⭐（2/5）

**做得好的部分：**
- 北凉地图有10个地点图标，美术风格已具备
- 地点和场景的绑定结构（locations.json）存在

**仅是骨架的部分：**
- 10个地点中，只有2个有场景绑定（清凉山王府、芦苇荡渡口）
- 地图探索（导航到不同地点、发现隐藏场景）尚未实现
- WorldMapScreen 存在但功能层面几乎是装饰性的

### 2.8 成就系统：⭐（1/5）

- achievements.json 存在定义
- SaveData 中有 achievements_unlocked 字段
- 但实际没有任何成就触发逻辑

## 3. 玩法趣味性分析

### 3.1 策略深度

**当前策略维度（低）：**

决策主要集中在：
1. 哪个场景派哪个角色（根据属性匹配）
2. 要不要装备某件装备

由于开局发全卡，"选派哪个角色"的答案通常是"属性最高的那个"，几乎没有组合价值。骰检主要用 max 模式，意味着多派人只有极少提升。

**潜在策略空间（未开采）：**
- sum 模式下的团队协作选择
- 装备分配的最优化问题
- 时间管理（本天该优先处理哪个场景？）
- 失败风险与预期收益的权衡

**评分：策略深度 2/10**

### 3.2 随机性

**当前随机来源：**
- 骰子点数（主要随机）
- 支持种子（seed）固定随机序列，即可复现

**随机体验感受：**
- 由于骰池计算多用 max 模式，强角色（属性高）的胜率基本锁定在一个较高区间
- 但目标值偏高（5-6个成功）导致即使强角色也有30-40%的失败率
- 随机性更多像是"偶尔受罚"，而非"真正的不确定性决策"

**评分：随机性体验 4/10**（有随机但缺乏玩家对随机的影响力）

### 3.3 重玩价值

**当前重玩变量：**
- 骰点结果不同（主要）
- scene_006、scene_007 有多条路线
- 难度调整（天数/金币不同）

**重玩价值低的根本原因：**
- 开局卡池完全相同，没有构筑差异
- 场景池极小（7个），很快读完所有内容
- 只有2种结局，没有"这局试试走另一条路线"的动力
- 无 meta 目标（图鉴、隐藏结局、人物收服）

**评分：重玩价值 2/10**

### 3.4 玩家代入感

**做得好的部分：**
- scene_006、scene_007 的叙事质量高，角色对话有IP味道
- 人物属性与原著人设高度吻合，让玩家有"在用原著角色"的感觉
- 结局标签（如温华获得"知己"/"损友"/"债主"）让玩家感受到关系差异

**代入感弱的原因：**
- 徐凤年是旁观者视角，而非真正可操纵的主角
- 大多数事件与玩家"输掉了什么/真正付出了代价"的连接感弱
- 失败只是扣数值，不是叙事意义上的"角色受到了什么"
- 苏丹卡的存在感很低，"处刑日"没有制造压迫感

**评分：玩家代入感 5/10**（叙事内容有情感，但系统缺少代入支撑）

## 5. 升级建议（优先级排序）

以下建议按"实现复杂度"和"玩法提升量"综合排序，每条说明：改什么、为什么能提升趣味性、实现复杂度（低/中/高）、参考游戏案例。

---

### P0-1：重做开局发牌逻辑，建立真正的成长弧度

**改什么：**
- 开局只给主角（徐凤年）+ 1-2名基础同伴 + 基础装备1件
- 其余角色通过剧情事件招募，或通过声望/金币解锁
- 真正使用 `DIFFICULTIES.initial_cards` 控制初始卡数

**为什么能提升趣味性：**
- 收集角色的过程本身就是成就感来源
- 让角色稀缺，每张卡有"得之不易"的珍视感
- 让角色之间的组合产生意义（只有3张角色时，哪个派去哪个场景是真正的决策）
- 解锁洪洗象或李淳罡时，玩家会有强烈的"我变强了"体验

**实现复杂度：低**
- 只需修改 TitleScreen 的 startNewGame 调用，传入筛选后的初始卡列表
- 配合写 2-3 个"招募事件"场景即可

**参考游戏案例：**
- 《Slay the Spire》：每局从少量卡牌开始，通过战斗奖励逐步扩展卡组
- 《Yes, Your Grace》：初始只有少量选项，随游戏发展解锁新角色和资源

---

### P0-2：把苏丹卡做成有处理流程的高压任务

**改什么：**
- 玩家在游戏中通过某个特定事件"被迫获得"苏丹之印
- 苏丹之印有以下几种处理方式，各自需要满足一定条件：
  1. **转嫁NPC**：找到特定人物，消耗声望/金币委托对方处理
  2. **特殊任务销毁**：完成某个高难度场景（需特定角色+高属性）
  3. **用作抵押**：某些政治场景中，可以"利用"苏丹卡换取资源
  4. **超时被处刑**：以上全部失败则处刑
- 设计若干以"处理苏丹卡"为轴心的场景

**为什么能提升趣味性：**
- 苏丹卡当前是纯粹的"持有=输"标签，毫无玩法张力
- 把苏丹卡变成需要主动处理的"烫手山芋"，每天的决策都有了紧迫性
- 多种处理方式增加策略多样性（高风险销毁 vs 低效但稳定的转嫁）
- 参考《Reigns》中"卡牌持有超时则游戏结束"的张力设计

**实现复杂度：中**
- 设计3-5个"处理苏丹卡"专属场景（有特殊unlock_conditions：持有苏丹卡）
- 实现一个"转嫁"效果（cards_remove sultan，给NPC cards_add）
- 调整游戏开始时的苏丹卡获取时机（不是开局直接给，而是通过事件）

**参考游戏案例：**
- 《The Riftbreaker》：紧急任务机制，玩家必须在限定时间内处理
- 《苏丹的游戏》（原作）：每张命令卡都需要在期限内完成特定动作

---

### P0-3：打通商店经济闭环，建立金币消耗口

**改什么：**
- 实现 `executeTrade()` 的完整逻辑：玩家可以用金币购买 shop_inventory 中的卡牌
- 商店支持：查看库存、买入、卖出（部分卡可出售换金）
- ShopScreen 展示真实可交易的物品列表
- 增加金币的持续消耗口：如角色每日维护费（可选）、招募场景需要付费

**为什么能提升趣味性：**
- 当前金币用处单一（只在少数场景结算里增减），中期就会堆积到无意义
- 商店打通后，玩家有"要不要花钱提升战力"的权衡
- 经济压力会让资源管理成为真正的玩法（花钱 vs 留钱应对风险）

**实现复杂度：低-中**
- 后端：SettlementExecutor.executeTrade() 实现实际买卖逻辑
- 前端：ShopScreen 接入真实库存，支持点击购买

**参考游戏案例：**
- 《Hades》：店铺经济是核心循环，每次挑战都有"用宝石买什么"的决策
- 《Yes, Your Grace》：资源短缺感让每个花费都有代价

---

### P0-4：把 unlock_scenes 接通，让事件形成内容网络

**改什么：**
- 在 `EffectApplier.apply()` 中处理 `effects.unlock_scenes`（调用 sceneManager 解锁）
- 设计以场景解锁为核心的事件链：
  - 招募温华（scene_006）→ 解锁温华专属场景
  - 与李淳罡结识（scene_007）→ 解锁剑道进阶场景
  - 声望达到一定等级 → 解锁宫廷深处的高风险场景
- 让地图上的地点随游戏进展逐步"开放"

**为什么能提升趣味性：**
- 当前所有场景都是固定的，玩家只能被动接受
- 场景解锁网络让玩家感受到"我的决策改变了世界"
- 招募某个角色打开新剧情线，重玩时有"这次试试不招募温华，走另一条线"的欲望
- 这是重玩价值最高效的提升方式之一

**实现复杂度：低**（几行代码，主要工作量是设计解锁链）
```typescript
// 需要添加的代码（EffectApplier.apply 中）
if (effects.unlock_scenes) {
  for (const sceneId of effects.unlock_scenes) {
    this.sceneManager.activateScene(sceneId);
  }
}
```

**参考游戏案例：**
- 《Hades》：首次死亡触发剧情，剧情推进解锁新房间/敌人/结局
- 《Disco Elysium》：某个技能检定成功，解锁后续对话分支

---

### P0-5：用失败状态替代纯数值扣减

**改什么：**
- 场景失败时，除了扣金币/声望，还应产生具体的"叙事状态"：
  - 战斗失败 → 角色获得"伤势"标签（降低下一次参与战斗类场景的属性）
  - 宫廷出糗 → 获得"丑闻"标签（特定场景失去解锁资格）
  - 债务未还 → 获得"欠债"标签（后续遭遇讨债事件）
- 这些状态标签通过后续场景可以消除
- 失败的代价变成"叙事债务"，而非单纯的数值惩罚

**为什么能提升趣味性：**
- 当前失败=扣钱/扣声望，玩家感受是"随机惩罚"而非"我做了错误决策"
- 叙事状态让失败变成"故事分岔"，有情感投入
- 状态积累让玩家感受到"这局角色经历了什么"，增强代入感
- 参考《Darkest Dungeon》：角色因恐惧获得quirk，影响后续表现

**实现复杂度：低-中**
- tags_add 机制已存在，只需在场景设计中使用（策划工作为主）
- 需要新增若干"负向标签响应场景"

**参考游戏案例：**
- 《Darkest Dungeon》：失败赋予角色quirk，影响长期表现
- 《Cultist Simulator》：后果不是数值，而是叙事卡牌生成

---

### P1-1：至少将一个属性维度发展为"政治资源"

**改什么：**
- 将声望从"解锁门槛值"发展为真正有游戏意义的资源：
  - 声望不同等级，影响不同类型事件出现（高声望才有高级宫廷事件）
  - 声望可以主动消耗（如给某事件"押上声望"降低难度）
  - 声望有事件性的上下波动（某些关键事件后公开评价变化）
- 可选：新增"江湖义气"作为第三资源，区分庙堂线vs江湖线

**为什么能提升趣味性：**
- 当前资源只有金币和声望，但两者感觉差不多（都是数值±）
- 声望有政治意义后，玩家才会在"花声望换短期利益"时感到真实的权衡
- 《雪中悍刀行》IP的核心张力就是"庙堂"和"江湖"两线，资源分化能映射这种张力

**实现复杂度：中**
- 需要在场景设计层面为声望赋予更多用途
- 可能需要对 PlayerState 增加新资源字段

**参考游戏案例：**
- 《Crusader Kings III》：声望、虔诚是独立的、具备明确用途的政治资源

---

### P1-2：给角色设计"功能位"（Role），而非单纯属性竞争

**改什么：**
- 每个角色增加1-2个独特机制标签，影响其适用的场景类型：
  - 徐凤年（protagonist）：宫廷类场景可用魅力替代部分社交；每场景增加1次重投
  - 老黄（mentor）：战斗类场景可触发"护主"效果，承担一次失败后果
  - 洪洗象（mage）：法术/卜算类场景可提前预知部分骰检目标值
  - 温华（swordsman+wanderer）：江湖场景额外bonus；宫廷场景有惩罚
- 通过场景设计，让角色的"功能位"比单纯属性值更重要

**为什么能提升趣味性：**
- 当前"拿属性最高的角色"几乎是最优解，构筑深度为零
- 功能位让"谁适合做什么"成为真正的决策
- IP人物原有的人设差异可以直接转化为游戏规则差异

**实现复杂度：高**
- 需要扩展 CardInstance 添加特殊能力系统
- 需要 SettlementExecutor 在计算时查询角色特殊能力

**参考游戏案例：**
- 《Into the Breach》：每个机甲有明确的战术角色，而非单纯比数值
- 《Fire Emblem》：职业系统让不同角色有不同战场价值

---

### P1-3：增加多结局结构（至少4种有差异的结局）

**改什么：**
- 在现有2种结局基础上扩展：
  1. **生存线**：成功处理苏丹卡，普通结局（已有）
  2. **北凉主线**：完成若干北凉相关任务链，触发特殊剧情胜利
  3. **江湖逍遥线**：收服若干江湖高手（李淳罡等），达成特定关系等级
  4. **执行失败**：被处决（已有）
  5. **角色结局**：某个角色的死亡/离开触发的分支结局
- 在 GameManager.checkGameEnd() 中扩展判断逻辑

**为什么能提升趣味性：**
- 当前只有"赢了或输了"，没有重玩动力
- 多结局让每次游戏都有不同的追求目标
- "这次走江湖路线会发生什么？"的好奇心是重玩的核心驱动

**实现复杂度：高**
- 需要设计完整的主线事件链
- 需要扩展结局判断逻辑和结局画面

**参考游戏案例：**
- 《Hades》：正式结局需要完成多次循环，且有额外隐藏结局
- 《80 Days》：不同路线触发截然不同的结局

---

### P2-1：引入Roguelite元素（随机初始配置）

**改什么：**
- 每局游戏开始时，从一个大卡池中随机选取初始角色+初始事件池
- 可以有"起源"选择（如"北凉世子"/"江湖游侠"/"宫廷幕僚"），影响初始卡和资源
- 增加周目目标（如"不招募任何温华路线角色通关"）

**为什么能提升趣味性：**
- Roguelite 的核心就是"每局不同"，这是重玩价值的最高效来源
- 随机初始配置让玩家每次都要重新思考策略

**实现复杂度：中**
- 需要扩展卡池数据结构
- 需要在 TitleScreen 实现起源选择UI

**参考游戏案例：**
- 《Slay the Spire》：每局随机卡包、遗物、地图
- 《Hades》：每次逃跑的路线和道具组合不同
