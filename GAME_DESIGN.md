## 1. 产品定义

### 1.1 项目定位

本项目是一款以《雪中悍刀行》人物、地点与气质为核心题材的 **场景驱动叙事卡牌游戏**。  
玩家扮演处于北凉、江湖与庙堂交界处的核心行动者，通过：

- 派遣角色进入场景
- 进行选择与骰子判定
- 获得角色、装备、情报、关系与资源
- 推动人物相识、阵营变化与章节进展

逐步完成从“人物邂逅原型”到“多章节叙事产品”的体验闭环。

### 1.2 当前建议产品主轴

在正式定稿前，建议将产品主轴定义为：

> **以人物关系与剧情推进为核心，以场景投入和骰子判定为底层驱动的叙事卡牌游戏。**

这意味着：

- 重点不是复杂数值战斗，而是“人物、关系、选择、局势”
- 战斗、社交、奇遇都统一落在场景系统中
- 角色与地点是内容组织核心
- 商店、装备、声望、资源是中层支撑系统

### 1.3 设计边界

当前版本不追求：

- 大型开放世界
- 纯 Roguelike 数值构筑
- 复杂战棋或独立战斗子游戏

当前版本追求：

- 有连续章节结构
- 有鲜明人物关系
- 有可持续扩展的数据驱动场景框架
- 有明确的资源与成长闭环

## 2. 版本目标

### 2.1 近期目标（可玩初版）

做出一个包含以下能力的可试玩版本：

- 1 张主地图（北凉）

### 2.2 中期目标

- 扩展到多地图结构
- 增加角色成长与关系沉淀
- 扩展支线与人物结局
- 完善音频与 UI 体验

## 3. 玩家体验目标

### 3.1 核心体验

玩家应该感受到：

1. **进入《雪中悍刀行》的世界**
2. **与关键人物相识、交锋、结盟**
3. **做决定，而不是只看剧情**
4. **资源有限，必须取舍**
5. **每次推进都改变人物关系与后续事件**

### 3.2 节奏目标

每个游戏日/回合应当形成以下节奏：

1. 查看当前可参与场景
2. 选择地点与事件
3. 投入角色/物品
4. 结算与剧情推进
5. 获得新人物、标签、资源或代价
6. 解锁下一批剧情或地点

## 9. 核心玩法系统设计

## 9.1 核心循环

建议正式循环为：

1. 进入当前地图/地点
2. 查看可参与场景
3. 投入角色/装备/道具
4. 进行剧情与判定
5. 获得资源、角色、关系标签、解锁内容
6. 推进章节与地图开放

## 9.2 场景系统

当前场景系统作为核心内容容器，正式沿用：

- Scene
- Stage
- Narrative Node
- Settlement
- Branch

场景可分为：

- 事件场景
- 挑战场景
- 商店场景

## 9.3 判定系统

当前已确定的体验要求：

- 使用 `3d6 + modifier + 金骰 ≥ DC`
- 支持重投
- 支持金骰补正
- 不直接向玩家暴露过于硬核的底层计算细节

### 9.3.1 当前公式背景

用户已明确现阶段希望使用：

> `3d6 + round(3×log₃(Σ3^attr / 3^opponent)) + 金骰 ≥ DC`

正式数值平衡文档可在后续单独拆出。

## 9.4 商店系统

商店系统需要承担：

- 购买装备/情报/消耗品
- 出售部分手牌
- 货币回收
- 中层资源调度

## 9.5 声望系统

声望不应只作为数值，应承担：

- 场景解锁门槛
- 角色态度差异
- 高级地点开放
- 特殊政治/庙堂事件前置

## 9.6 关系系统

建议最小实现：

- 角色标签沉淀
- 角色关系标签
- 若干角色线分支

后续可扩展为：

- 好感度
- 阵营倾向
- 关系结局
