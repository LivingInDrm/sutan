# v2.6 prompts — production cut-in (B4-Lean template + v3_slot description)

**Date**: 2026-04-18
**Scope**: merge two separately-verified experiments into production code paths.
**Evaluation**: already done in v2.5 archive (B4-Lean template) and in the single-factor
desc_v2_vs_v3_5char run on 2026-04-18 (v3_slot description). No new eval in this cut.

## 1. What landed in production

### 1.1 Template (`tools/asset-manager/backend/templates.json`)

Replaced the four portrait-related keys with the B4-Lean variant:
- `style_base`        — semi-realistic wuxia painterly, rim/volumetric light, neutral-cool palette
- `no_text_constraint` — one-line directive
- `style_negative`    — added flat-shading / cel-shading / warm-cast / muddy-palette / muddy-face negatives
- `portrait_template` — removed pose+palette boilerplate, added explicit `FACE AND EYES` block
  (requires iris color, catchlight, lash detail)

`item_template` and `scene_template` are untouched. See `v2.5_prompts.md` for the
full diff and word-count delta vs B4.

### 1.2 Description prompt (`backend/services/characters.py`)

Replaced `_DESCRIPTION_SYSTEM_PROMPT` (the old ~750-word free-form instruction)
with the v3 slot-based prompt (~370 words). Changed the contract from "4 English
strings" to "4 objects with 9 slot fields each", and assemble deterministically
in-process.

Removed:
- `_load_character_refs` + `_TEST_SET_PATH` — description generation no longer
  reads `tests/portrait-eval/test_set.yaml`. Ground truth now relies on the LLM's
  own knowledge of 《雪中悍刀行》 plus the bio field. The character reference
  YAML block was dropped from the user message.

Added:
- `_assemble_slot_description(slot)` — pure function that concatenates
  `"{gender} character. {face} {eyes} {hair} {build}. Wearing {costume}, holding {prop}. {pose}, {mood}."`
- `_REQUIRED_SLOT_KEYS`, `_VALID_GENDERS` — schema constants.
- Fail-fast validation in `_generate_descriptions_blocking`: parse errors, non-array,
  <4 variants, missing/empty slot fields, or invalid gender all raise `ValueError`
  with the raw payload. No silent fallback.

Public contract unchanged:
- `_generate_descriptions_blocking(client, name, bio) -> List[str]` of length 4.
- Outer callers (`generate_description`, SSE stream, `/characters/ai-generate`) are
  not touched.

## 2. Why these specific changes

### 2.1 Nine-slot structure
The LLM now produces structured dicts. This gives us:
- **Byte-identical identity anchor** across the 4 variants (gender/face/eyes/hair/build
  are required to match), replacing the "ensure Part-1 sentence is identical"
  rule that the old free-form prompt enforced only behaviorally.
- **Enforced variant axes** (costume/prop/pose/mood must differ), replacing the
  weaker "Part-2 must differ" English instruction.

### 2.2 Required `gender` field
Old free-form outputs routinely omitted explicit gender words ("slender figure
with jade hairpin…"), which leaked into ambiguous portraits. Making `gender`
a mandatory slot, validated against `{male, female}`, and prepending
`"{gender} character."` to the assembled description makes the final image
prompt's gender signal unambiguous.

### 2.3 Proper-noun counter-examples in the prompt
Added explicit "NEVER literally translate" rules for four known-problematic
names (风马牛 / 梅子酒 / 老黄 / 绣冬 / 春雷). Previously 老黄 occasionally
rendered as "old yellow" or a jaundiced old man.

### 2.4 Dropped novel_refs injection
Per user requirement ("不依赖 refs"). The test_set.yaml excerpts were both
- **expensive** to curate per-character, and
- **redundant** for well-known characters from a widely-known novel.

The LLM already knows the major 雪中悍刀行 cast. For unknown or homebrew
characters the `bio` field still flows through.

  - 陈芝豹 → `male character`, "long spear" prop (not "plum wine" literalization)
  - All outputs English-only, all 4 variants share byte-identical anchor.

  (`test_character_deploy`, `test_character_generation_stream`) all pass.
  `test_item_deploy` failure is a pre-existing fixture-isolation bug in the
  `test_item_deploy` suite and is unrelated to this cut (reproduces on HEAD
  without our `characters.py` change).
